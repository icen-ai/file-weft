# FlowWeft 1.0.0（开发中）

状态：**未发布，禁止作为稳定版本消费。**

本文件预留 1.0 发布合同，以便 `1.0.0-SNAPSHOT` 开发线持续验证制品内容。当前
稳定版本仍为 `0.0.3`；在 [1.0 交付总账](../flowweft-1.0-delivery-ledger.md)
全部关闭、受保护标签流水线成功且远端制品完成匿名冷读前，不得把本文件改写成
已发布声明。

当前接线库存是 32 个实体 JAR：完整保留 `0.0.3` 已发布的 19 个 `fileweft-*`
坐标，并新增十三个从未发布过的 `flowweft-retrieval-api/spi/runtime`、
`flowweft-agent-api/runtime`、`flowweft-workflow-api/spi/domain/runtime/persistence-jdbc`、
`flowweft-migration-cli`、`flowweft-adapter-dify` 与 `flowweft-adapter-oss` 坐标。这只是
开发期库存，不是 1.0 最终模块数；最终发布必须逐坐标列出 artifact kind、Java
基线、class ownership、POM 与 Gradle Module Metadata 消费证据。更名本身不产生
虚构的旧坐标别名或重复 class JAR。

产品范围与兼容决策见
[ADR 0001](../decisions/0001-flowweft-1.0-product-scope.md)。最终发布说明必须
记录精确 commit、tag、CNB SN、每条发布 lane、制品/镜像清单、签名、provenance、
升级矩阵、已知限制和支持/弃用日期。

## 开发期依赖安全合同

当前 1.0 接线把 Boot 3 Web Starter 的 Tomcat embed core/el/websocket 固定为
`10.1.57`，Logback classic/core 固定为 `1.5.38`；这五项既作为 Gradle Module
Metadata 的严格约束，也作为 POM 的直接 runtime 依赖发布。这样普通 Gradle
platform 与纯 Maven POM 消费者都会解析到同一组修复版本，且不会重新引入 Spring
Boot 已排除的 `tomcat-annotations-api`。Gradle 消费者若用旧 Boot BOM 的
`enforcedPlatform` 强制降级，GMM 门禁必须因版本冲突而失败；Maven 宿主若自行在
根 dependencyManagement 强制旧版，也必须由宿主的 Enforcer/依赖树门禁拒绝，不能
把“宿主主动覆盖”解释为 FlowWeft 已验证组合。

迁移 CLI 的开发期运行图要求 PostgreSQL JDBC `42.7.11`、MySQL Connector/J
`9.7.0` 与 protobuf-java `4.31.1`，并拒绝 Connector/J 的可选 OCI SDK 进入闭包。
以上版本只有在重生锁文件和依赖校验、真实 PostgreSQL/MySQL、Dev 验收、POM/GMM
冷消费者及 runtime-closure OSV 门禁全部通过后，才可成为正式发布证据。Boot 2、
Boot 3.5 与后续 Boot 4 的长期支持等级仍需单独 ADR 拍板，本节不提前作支持承诺。
