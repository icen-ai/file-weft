## 撤回原因与升级边界

- 该版本使用过共享的 `classpath:db/migration` 和默认 `flyway_schema_history`，可能与宿主迁移的版本号、脚本或 checksum 冲突。
- 该版本早于后续的专属迁移命名空间以及 V024 续传所有者隔离等安全修复，不能作为当前能力或安全基线。
- 当前 checkout 是 `0.0.2-SNAPSHOT` 开发线；其中的构建、安装和发布任务只会处理当前项目版本，不能用于重新生成或宣传 `0.0.1` 发布物。
- `0.0.2` 及后续版本不会自动收养旧共享 history。旧库必须先停止全部 API/Worker、完成可恢复备份，再由 DBA 核验实际 schema、成功版本、脚本名、checksum、安装顺序以及宿主同版本迁移。
- 禁止通过盲目 baseline、repair、复制或删除 history 行绕过校验。形成经评审的人工转换或数据迁移方案前，不得原地滚动升级。

当前接入坐标、构建方式和生产迁移要求以仓库根 `README.md` 与 `docs/production-operations.md` 为准。本文件不包含可执行的接入、发布或删除命令，也不证明任何外部副本已经删除。
