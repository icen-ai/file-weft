# FileWeft 0.0.1

发布日期：2026-07-13

正式 Maven 坐标：`ai.icen:*:0.0.1`

> 早期试推的 `com.fileweft:*:0.0.1` 已撤回并从当前 CNB 制品库清空。它与本次正式发布的 group、JVM 包名、数据库安全边界和制品内容均不同；任何本地或代理缓存中的旧坐标都不得用于接入。

FileWeft 0.0.1 是首个可公开使用的正式版本，采用 Apache-2.0 许可证，版权主体为 `icen.ai`，安全问题通过 `support@icen.ai` 私密接收。

## 正式命名空间

- Maven group：`ai.icen`
- JVM 包名：`ai.icen.fw`
- 17 个公共模块均发布主 JAR、sources JAR、POM 和 Gradle module metadata。
- POM 声明 Apache-2.0、项目主页、SCM 与维护者信息；JAR 和发布 ZIP 内置 LICENSE/NOTICE；正式 CycloneDX SBOM 精确描述 17 个发布模块及其 Apache-2.0 许可。

## 核心能力

- 分层架构：Core、SPI、Domain、Application、Persistence、Starter、Adapter、Doctor、Agent。
- Java 友好的 JDK 8 基线公共契约，以及 Spring Boot 2 / Spring Boot 3 对称 Starter。
- 强制租户隔离、可信身份与授权上下文、目录 ACL、版本与生命周期规则。
- PostgreSQL 持久化、Outbox/Task 租约、幂等请求、审批路由、审计和操作日志。
- S3/RustFS 存储、断点续传、提交结果未知恢复与所有者隔离。
- 多下游 REQUIRED/OPTIONAL 投递、部分失败、独立重试、撤回和发布代次围栏。
- 即时、异步与系统 Doctor，安全投影的插件清单和确认式 Agent 结果。
- 仓库内 `fw-dev` 双租户验收台、独立下游模拟器、PostgreSQL、RustFS、Nginx 与 Playwright 回归。

## 数据库迁移安全

- FileWeft 迁移只位于 `classpath:ai/icen/fw/db/migration`，历史表固定为 `fileweft_schema_history`。
- Starter 明确支持 `DISABLED`、`VALIDATE`、`MIGRATE`，默认 `DISABLED`。
- 新库先写入受校验的版本 0 命名空间标记，再从 V001 执行全部业务迁移。
- 缺失或篡改的标记、未来版本、旧共享 history 记录、无专属 history 的既有业务表都会失败关闭。
- Boot 2 与 Boot 3 在多 DataSource、lazy initialization 和宿主 Flyway 场景下保持相同安全边界。

## 旧试推升级边界

旧 `com.fileweft:*:0.0.1` 可能曾使用共享 `classpath:db/migration` 与默认 `flyway_schema_history`。正式 `ai.icen:*:0.0.1` 不会自动收养这些记录。

旧库升级必须停止全部 API/Worker、完成可恢复备份，并由 DBA 核验 schema、成功版本、脚本名、checksum、安装顺序和宿主迁移。禁止使用盲目 baseline、repair、复制或删除 history 行绕过校验。完整步骤见 [生产部署与恢复](https://cnb.cool/china.ai/file-weft/-/blob/main/docs/production-operations.md)。

## 已知边界

- 当前唯一经过真实数据库迁移和并发测试的持久化目标是 PostgreSQL；MySQL 8 尚未实现。
- 当前提供通用 SPI、S3 与本地基线；OSS、Dify、ESE、AppBuilder 等厂商适配器需要独立真实环境与合约测试。
- 历史保留/归档、生产 SLO、RPO/RTO 与容量目标必须由实际部署方定义并验证。
