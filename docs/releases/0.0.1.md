# FileWeft 0.0.1

发布日期：2026-07-12

`0.0.1` 是 FileWeft 第一版可供项目集成的正式基线。所有公共库制品使用统一坐标：

```text
com.fileweft:<module>:0.0.1
```

## 推荐接入

Spring Boot 3（JDK 17+）：

```kotlin
dependencies {
    implementation("com.fileweft:fileweft-spring-boot3-starter:0.0.1")
    implementation("com.fileweft:fileweft-web-spring-boot3-starter:0.0.1")
}
```

Spring Boot 2（JDK 8+）：

```kotlin
dependencies {
    implementation("com.fileweft:fileweft-spring-boot2-starter:0.0.1")
    implementation("com.fileweft:fileweft-web-spring-boot2-starter:0.0.1")
}
```

不需要正式 HTTP 层时，只引入对应的 `fileweft-spring-boot*-starter`。插件和适配器开发可按需引入 `fileweft-spi`、`fileweft-testkit` 或具体 adapter 模块。

在仓库中生成可共享 Maven 仓库与发布包：

```powershell
.\gradlew.bat releaseBundle
```

输出：

- `build/repository/`：完整 Maven 仓库，可上传到 Nexus、Artifactory、CNB 制品库或作为本地文件仓库使用。
- `build/release/fileweft-0.0.1-release.zip`：Maven 仓库、CycloneDX SBOM、README 与本说明。

同机项目需要立即试用时：

```powershell
.\gradlew.bat installReleaseToMavenLocal
```

然后在消费项目启用 `mavenLocal()`。

发布到本项目的 CNB Maven 制品库时，只通过环境变量或用户级 Gradle 属性传入令牌，禁止把真实令牌提交到仓库：

```powershell
$env:CNB_TOKEN = "<具备制品库权限的访问令牌>"
.\gradlew.bat publishCnbArtifacts --no-configuration-cache --no-parallel --no-daemon
```

`publishCnbArtifacts` 会先执行完整 `releaseCheck`，且拒绝在配置缓存开启时载入发布凭据。制品库地址为 `https://maven.cnb.cool/china.ai/maven/-/packages/`，用户名固定为 `cnb`。消费项目使用相同仓库地址，并通过自己的安全凭据读取 `com.fileweft:*:0.0.1`。

## 本版能力

- 多租户文档、版本、文件对象和宿主目录绑定。
- 角色授权、用户 ID/名称快照、审计与操作 Trace。
- 单人审批、双人会签、驳回修订及发布生命周期。
- 多下游必达/可选交付、Outbox、重试、撤回与人工恢复。
- PostgreSQL/Flyway 持久化及 RustFS、MinIO 等 S3 兼容存储。
- 任务与 Outbox token 租约围栏，防止失租 Worker 覆盖新结果。
- 即时、异步和系统 Doctor 正式 v1 API，响应脱敏并支持目录 ACL。
- Boot 2、Boot 3 Starter，以及 Java 8/11/17/21/25 兼容矩阵。
- 双租户 Dev 验收台、Compose API 回归和 Playwright 浏览器回归。

## 已验证门禁

- 全仓 `check`。
- `compatibilityCheck` 多 JDK 运行时矩阵。
- `verifySbom` CycloneDX 校验。
- PostgreSQL 16 实库迁移、并发、隔离和原子性测试。
- PostgreSQL + RustFS + API/Worker + 模拟下游的 `fw-dev` Compose 验收。
- 正式 v1、角色控件、双租户、审批、同步、Doctor 与断点续传浏览器验收。

## 明确边界

- 本版唯一完成实库迁移与并发验证的数据库是 PostgreSQL；MySQL 不是 `0.0.1` 的受支持目标。
- S3 兼容存储是已验证基线；OSS、CenterFile、Dify、ESE、AppBuilder 等厂商实现应作为独立适配器或插件交付。
- 生产宿主仍须提供 `TenantProvider`、`UserRealmProvider`、`AuthorizationProvider` 及真实凭据配置。
- 发布包当前未做密码学签名。上传公共制品仓库前，应由项目所有者确定许可证、安全披露入口、签名密钥和目标仓库。
- 发布包中的 CycloneDX 是用于整仓安全审计的全仓 SBOM，包含不发布的 `fileweft-dev`；它不是逐个 Maven 制品裁剪后的 SBOM。
