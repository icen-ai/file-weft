# FileWeft 0.0.2

状态：开发中，尚未发布。

当前稳定版本仍为 `ai.icen:*:0.0.1`。本文件只记录 `0.0.2-SNAPSHOT` 开发周期中已经合入并通过验证的变更；在正式发布门禁、远端制品回读与独立消费者验收全部完成前，不得将其描述为可用版本。

## 开发中变更

- 发布仓库校验器从受限解析的 Maven version metadata 重构 timestamp/build number，验证四类制品与 17 个模块使用同一 SNAPSHOT 身份；不再错误假定 SNAPSHOT 制品文件名包含字面量 `-SNAPSHOT`。
- 新增受权工作流决策证据读取：`GET /fileweft/v1/documents/{documentId}/workflow-decisions` 同时要求 `document:audit` 与 `document:read`，并遵守可信租户、文档可见性和目录 ACL；原 `/workflows` 历史继续保持身份脱敏。
- 新审批或驳回在任务上不可变保存决策时的操作者 ID、可选安全名称快照与 `decidedTime`。V026 不从当前受理人或审计记录猜测遗留决策者，历史完成任务以 `decisionEvidenceRecorded=false` 和空证据字段表达 `UNKNOWN`/未记录。
- 宿主用户 ID 统一为最多 256 个 UTF-16 code unit 的不透明安全字符串；`Long`、`Int`、UUID 与外部目录 ID 由宿主稳定转为字符串。审计、操作日志、Agent 确认人与工作流受理人/决策人列同步扩宽为 `varchar(256)`。
- 工作流任务 upsert 增加租户、工作流、受理人和既有决策状态围栏，拒绝跨租户任务 ID 碰撞，并保持首个已提交决策的操作者与时间不可变。
- Dev 验收前端按权限读取并显示决策证据，覆盖普通历史脱敏、无权限不请求、双人会签不同操作者和遗留任务未知状态。
- 正式 CycloneDX SBOM 改为 17 个发布模块各自独立的生产 `runtimeClasspath` 并集闭包，禁用 Dev 与 Gradle build environment，按可达性剔除测试/编译孤儿；真实输出不再包含 Mockito、AssertJ、Kotlin compiler 或 Boot starter-test，同时保留 TestKit 合法的 JUnit 运行时。

## 兼容与迁移

- 不修改任何已经发布的数据库迁移；新增数据库能力只允许使用更高版本的只前进迁移。
- V026 是加法扩列与身份列扩宽迁移。生产升级先运行 `docs/sql/postgresql-v026-workflow-decision-evidence-preflight.sql` 并处理全部不安全宿主用户 ID，再停止所有旧审批入口节点、等待在途事务并重跑预检后迁移；新旧审批节点不可混跑。
- 回滚应用时保留 V026 列、约束和已经写入的决策证据，不得缩窄用户 ID 列、清空操作者快照或把遗留任务推断为某个用户。
- `0.0.1` 的 Maven 坐标、源码标签和发布 ZIP 保持不可变。
