受监管团队需要的不仅是文档存储，还需要审计轨迹。FlowWeft 会记录文档级别的事件，例如生命周期转换、工作流决策、交付尝试和同步结果。本页介绍如何查询这些轨迹，同时不破坏租户隔离，也不把身份信息泄漏到指标中。

## 1. 记录哪些事件

审计日志只捕获 FlowWeft 自身产生的事件，不包含宿主业务代码。典型条目包括：

| 事件类别 | 示例 |
| --- | --- |
| 生命周期 | 创建草稿、提交审批、发布、下线、归档、恢复。 |
| 工作流决策 | 任务通过、任务驳回。 |
| 交付 | 连接器同步成功、同步失败、移除重试请求。 |
| 任务 | OCR 开始、OCR 成功、OCR 永久失败。 |
| 系统 | Doctor 检查运行、清理完成。 |

从 FlowWeft 角度看，日志是只追加的。如果需要更丰富的业务上下文，宿主应把领域事件发到自己独立的审计存储，而不是直接写入 FlowWeft 审计表。

## 2. 查询文档日志

稳定端点返回单篇文档的按时间排序列表：

```bash
curl http://localhost:8080/fileweft/v1/documents/DOC-001/logs
```

响应遵循 FlowWeft 标准 v1 统一外层，并包含该文档的事件列表。使用 traceId 与宿主网关日志关联。

> [!NOTE]
> 具体字段名和排序以 0.0.2 响应契约为准。当前格式请参见 [HTTP API 参考](../reference/http-api.md)。

## 3. 读取工作流决策

对于经过审批的文档，专用 workflow-decisions 端点展示每个审批任务的结果：

```bash
curl http://localhost:8080/fileweft/v1/documents/DOC-001/workflow-decisions
```

受权工作流决策证据视图需要同时具有 `document:read` 和 `document:audit` 权限。0.0.2 中新记录的审批与驳回返回不可变操作者 ID、可选安全显示名快照和 `decidedTime`；遗留记录显式表示未保存该证据。普通工作流历史继续保持身份脱敏。

## 4. 租户与隐私边界

审计查询始终由 `TenantProvider` 解析的当前租户隔离。跨租户访问在 SPI 层被拒绝，而不是靠请求参数过滤。

FlowWeft 指标刻意排除高基数或敏感标签：

- 计数器和 Gauge 不带 `tenantId` 标签。
- 不带文档 ID 标签。
- 不带用户 ID 标签。

如果需要按租户或按用户的审计仪表盘，请从正式审计端点或宿主自己的审计存储构建，而不是从指标中反推。

> [!WARNING]
> 永远不要信任客户端通过查询参数传来的 `tenantId`。租户必须来自经过认证宿主上下文的 `TenantProvider` SPI。

## 5. 保留与归档

FlowWeft 把审计记录与文档元数据放在同一持久化 schema 中。保留策略由宿主决定：

1. 根据合规要求确定审计行保留时长。
2. 删除前把旧记录导出到冷存储。
3. 除非策略明确允许，否则不要在引用的文档仍活跃时删除审计行。

## 常见问题

**Q：我可以把自定义事件写入 FlowWeft 审计日志吗？**
不可以。FlowWeft 只记录自己的事件。宿主业务事件应进入你自己的审计流水线。

**Q：登录失败会记录在这里吗？**
不会。认证是宿主职责，请在身份提供方或网关层记录认证事件。

**Q：审计日志包含原始文件内容吗？**
不包含。它只包含元数据、操作和结果，从不包含文件字节。
